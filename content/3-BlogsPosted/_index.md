### <a href="3.1-Blog1/" >Blog 1 - SESSION POLICIES IN AMAZON EKS POD IDENTITY</a>
This blog introduces the newly added session policies feature in Amazon EKS Pod Identity, which allows you to narrow IAM permissions flexibly and precisely for each pod without needing to create multiple separate IAM roles. This is an important step forward that helps apply the principle of least privilege more effectively in large-scale Kubernetes environments.

### <a href="3.2-Blog2/" >Blog 2 - PROD-READY ARCHITECTURE: BUILDING AN AI GATEWAY TO AMAZON BEDROCK WITH AMAZON API GATEWAY</a>
This blog shares practical experience in designing and deploying an AI Gateway using Amazon API Gateway and AWS Lambda to govern, control costs, and authenticate access to Amazon Bedrock.

### <a href="3.3-Blog3/" >Blog 3 - A POWERFUL FEATURE OF AWS LAKE FORMATION: UNIFYING DATA ACCESS</a>
This blog introduces the new feature in AWS Lake Formation that grants temporary credentials for EMR Spark to access S3 data directly, simplifying data access control and eliminating Access Denied errors.
