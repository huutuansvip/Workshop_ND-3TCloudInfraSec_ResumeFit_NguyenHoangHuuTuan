# Summary Report: "FCAJ Community Day"
*(FCAJ Community / AWS Study Group Gathering)*

### Event Objectives
* **Program Goal:** Far from being just a place to listen to presentations, the event serves as a space to connect, inspire, and encourage everyone to actively start conversations to find new friends or partners.
* **Core Messages:** Covered 6 sharing sessions spanning from AI and Cloud to real-world experiences. Key focus was on adapting to job market shifts in the AI era, AWS architecture (CloudFront, Amazon Q), writing contextual prompts, LLM non-determinism, Hackathon experiences, and enterprise-grade Multi-Agent system design.
* **Value for Attendees:** Provides practical and realistic insights from experts directly building large-scale systems; understanding how to design and operate secure, reliable systems that address actual user needs.
* **Organizer:** FCAJ Community (AWS Study Group).
* **Location:** 26th Floor, Bitexco Tower, 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City.
* **Role in Event:** Attendee.

### Speakers
The event featured multiple speakers sharing practical and inspiring topics:
* **Quynh Mai:** MC.
* **Nguyen Gia Hung:** Head of Solution Architect at AWS Vietnam, Founder of FCAJ.
* **Speaker 1 - Tinh:** Platform Engineer at Gotam X.
* **Speaker 2 - Hai Anh:** Working at Pacific Vietnam (Formerly presented at AWS Singapore Summit, Silicon Valley).
* **Speaker 3 - Thinh:** DevOps Engineer.
* **Speaker 4 - UTM Group:** Developers (Colleagues sharing their Hackathon project).
* **Speaker 5 - Duc:** LLM Optimization expert, discussing large language model engineering.
* **Speaker 6 - Vy:** Currently working at VBBank (VPBank), discussing Enterprise Multi-Agent systems.

### Key Highlights

#### 1. Problem Overview
* The IT industry is anxious about rapid AI development.
* Engineers often write prompts without context, resulting in hallucinations.
* Difficulties in managing sudden cloud cost spikes (bill spikes).
* LLMs return different results even with temperature fixed at 0.
* Lack of financial report data when calculating credit scores for startup businesses.

#### 2. Solutions Introduced
* Build actual products to gain recognition rather than just doing demo exercises.
* Provide domain-specific business contexts to AI instead of generic queries.
* Use CloudFront's flat-rate pricing to optimize infrastructure costs and prevent DDoS attacks.
* Deploy Multi-Agent systems to delegate specialized tasks instead of forcing a single AI model to do everything.

#### 3. Technologies / Services / Tools
* **AWS Services:** Amazon Q (QuickSight), Amazon CloudFront (OAC, Shield, WAF).
* **AI & Tools:** CrewAI, Bedrock Agent Core, LLM parameters (Temperature, Logits, Soft max, Arg max), Model Context Protocol (MCP).
* **Infrastructure as Code:** Terraform.

#### 4. Demo / Case Study
* Demo of Amazon Q importing Excel financial reports to generate analytical dashboards.
* Los Hackathon case study with "UTM Morpo" - automatically generating UI layouts from wireframe images and allowing users to drag-and-drop or edit code directly.
* Demo hosting 2 LLM models (one on AWS Bedrock, one locally) with temperature = 0 to compare output consistency.
* Case study of building a Multi-Agent credit rating system for startups in a banking environment.
* **Key Takeaway:** The "Internet Builder" syndrome - the habit of dragging tools, rules, or code found online into a project without understanding if it fits the enterprise environment. Extremely strict security protocols in banks (security will confiscate laptops if unauthorized MCP servers are installed).

### Key Takeaways

#### 1. Design Mindset & Methods
* AI will not replace humans, but it will create a new stream of work for professionals who fix and maintain products generated by AI.
* When designing systems, always answer: Who uses it, what do they use, why do they use it, and when do they use it.

#### 2. Technical Knowledge
* Understand the role of Platform Engineering (creating Internal Developer Platforms (IDP) to let Devs provision resources).
* Understand LLM inference optimization techniques, explaining why output results are never 100% identical.

#### 3. Best Practices
* To get high-quality AI outputs, instead of cramming information, establish proper knowledge transfer by extracting business logic from experts.
* Rotate API keys and IAM Access keys frequently to prevent huge budget loss.
* Set up a dedicated validation Agent to filter and challenge output results from other AIs.

#### 4. Practical Experience
* Copying and pasting code from AI without understanding coding standards can cause major bugs in Production.
* Within an enterprise, whoever makes decisions based on AI outputs is fully legally liable for any errors; one cannot blame the AI.

### Applying to Work
* **Immediate Application:** Refine prompt templates (contexts) to be closer to the team's domain and workflows. Adjust LLM parameters (e.g., temperature = 0.1 instead of 0) to avoid repetitive phrasing.
* **Tech to Test:** Use Terraform for Infrastructure as Code (IaC) instead of manual clicks, making rollback and scaling easier. Test Amazon Q as an assistant to summarize meetings and automatically send summaries to Teams/Email.
* **Workflow Improvement:** Build Multi-Agent workflows with clear task division (e.g., one for Research, one for Risk Assessment, one for Review). Draft ROI (Return on Investment) assessments before proposing any AI project to leadership.

### Event Experience
* **Learning from Speakers:** Gained highly practical advice from engineers working on large-scale systems, highlighting the gap between university projects and enterprise software.
* **Hands-on Exposure:** Observed live demonstrations of voice-to-dashboard creation and real-time editing of generated web layouts.
* **Networking & Connection:** Friendly speakers who encouraged confidence and interactive Q&A sessions.
* **Most Impressive Part:** Honest discussions about how recruiters look for actual shipped products rather than grades, and the ChatGPT code-copying incident that caused production issues at VBBank.

### Lessons Learned
* **Most Important Knowledge:** Software Engineering fundamentals are key to shipping AI models safely and scalably to production. Degrees and traditional skills remain crucial for passing resume screens.
* **Career Lessons:** "Garbage in, Garbage out" - if the input data is low-quality, the AI output will be too. When overthinking with too many ideas, the best approach is to trim down and focus on shipping a single core feature.
* **Next Steps:** Strengthen AI mindset and adoption in business. Master a Cloud provider (like AWS) along with security (Security, IAM) and Terraform instead of just learning superficial AI. Always remember three pillars: "secure", "reliable", and "user-focused".
